Core Philosophy

Built for the Real World

Designed for shops in areas with unreliable internet. Business never stops because connectivity does.

🔌

Zero Dependency

The desktop app boots, runs, and provides full utility forever without ever touching the internet.

☁️

Sync is a Feature

Cloud sync is an optional add-on for backup and multi-device access. Local SQLite is always the source of truth.

🧩

Modular "Lego" Design

A Core Kernel hosts pluggable Business Modules. Enable only what your shop needs — Repair, Retail, Scheduling.

📱

Mobile-First Mechanics

All mechanic-facing views are optimized for mobile phones with large touch targets and card-based layouts.

Technical Architecture

Three-Layer Stack

From local desktop to cloud swarm, each layer has a clear role.

Layer 1

🖥️ Local Kernel (Desktop/Tablet)

  • Tauri (Rust + React) desktop app
  • SQLite database — full offline capability
  • Core tables: users, sync_state, devices
  • Module tables loaded on demand
  • Kysely for type-safe SQL queries
Layer 2

📡 Scout Nodes (Mobile/Tablet)

  • Connect to Commander via LAN (no internet needed)
  • Mechanics enter job data on site
  • Push changes to Commander in real-time
  • Touch-optimized React views
Layer 3

☁️ Cloud Swarm (Optional SaaS)

  • Next.js management dashboard
  • Postgres (Supabase/Neon) mirror
  • Encrypted diff-based sync
  • Remote viewing for owners
  • Customer booking portal

Network Topology

Three Node Types

Each device plays a role in the local-first network. No single point of failure.

🖥️

Commander Node

The main PC/server for the shop. Holds the full SQLite database. All Scout Nodes sync to this.

📱

Scout Node

Mobile phones or tablets used by mechanics. Connect to Commander via LAN Wi-Fi. No internet required.

☁️

Cloud Node

Optional. Commander syncs encrypted diffs for remote backup, owner dashboards, and customer portals.
